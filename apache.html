Install

소스를 직접 빌드해서 설치한 것과 패키지 관리자를 이용해서 설치한 것의 차이

실행 스크립트 파일 위치

소스 설치 

$ find / -name apachectl
/usr/local/apache2/bin/apachectl

배포판 설치 - CentOS 5 

$ find / -name apachectl
/usr/sbin/apachectl

설정 파일 위치

소스 설치 

$ find / -name httpd.conf
/usr/local/apache2/conf/httpd.conf
/usr/local/apache2/conf/original/httpd.conf
/root/httpd-2.2.8/docs/conf/httpd.conf

배포판 설치 - CentOS 5 

$ find / -name httpd.conf
/etc/httpd/conf/httpd.conf

VirtualHost 설정

IP 주소 하나에 여러 개의 이름 기반 웹사이트를 운영하는 경우 설정 예. 

NameVirtualHost *:80		# Incoming IP: ANY, Incoming Port: 80

<VirtualHost *:80>
DocumentRoot /www/example1
ServerName www.example1.com
</VirtualHost>

<VirtualHost *:80>
DocumentRoot /www/example2
ServerName example2.org
ServerAlias *.example2.org lg.goldstar.com
</VirtualHost>
  • 가장 처음에 나오는 VirtualHost가 디폴트 VirtualHost
  • VirtualHost를 사용하는 경우 기존 DocumentRoot 설정은 무시.
  • VirtualHost 설정을 확인하는데 httpd -S 명령이 유용. (CentOS)
